Platform Adaptations

This UI Style Guide serves as the visual identity baseline and design token specification for Crewcation, a dual-platform collaborative travel framework. Rather than forcing rigid 1:1 pixel replication, the design system is tailored to the strengths of each platform, leveraging native iOS paradigms with SwiftUI and responsive web patterns for the Progressive Web App (PWA). Aligning these underlying primitives ensures visual harmony, platform-specific familiarity, and cohesive interaction states across all user viewports.

Platform Diversion: Colors

PWA / Web: Uses semantic CSS variables mapped to light/dark mode root selectors (e.g. --background, --foreground) which switch dynamic surface tints using Tailwind class modifiers.

iOS / Android: SwiftUI loads color tokens from native Assets.xcassets catalog. These resolve appearance changes natively from system settings without custom JavaScript toggle triggers.

Platform Diversion: Typography

PWA / Web: Imports Google Web Fonts Inter (for geometric headings) and Nunito Sans (for soft metadata and copy) via stylesheet injections.

iOS / Android: iOS defaults to Apple's system font San Francisco (SF Pro) to match native platform aesthetics, using SwiftUI text modifiers (e.g. .font(.system(.title, design: .default))) for dynamic type scaling.

Platform Diversion: Buttons

PWA / Web: Rendered using CSS button rules with explicit border-radius (e.g. rounded-[26px]) and active scales declared in index.css. Interaction states rely on standard hover/active selectors.

iOS / Android: Implements SwiftUI Button wrapping text in custom ButtonStyle structures. iOS handles tactile touch scaling and gesture bounds natively with haptic feedbacks.

Platform Diversion: Inputs & Forms

PWA / Web: Textboxes are standard HTML inputs/textareas with custom focus rings and warm grey colors. Keyboard overlay offsets and form validations are managed manually using React hooks.

iOS / Android: SwiftUI implements TextField and TextEditor, inheriting iOS system autofill rules, native field clear parameters, and automatic keyboard avoidance layouts.

PWA / Web: Rendered as custom flexboxes where active items shift position or styles via React state and CSS transitions (active item scale and shadow changes).

iOS / Android: Employs native Picker styled as .pickerStyle(.segmented), utilizing Apple's native slide transition and standard system tactile feedbacks.

Material Symbols & Icon Tokens

Rather than using custom inline vector assets, Crewcation uses standard Google Material Symbols Rounded font icons as semantic identifiers (switching properties based on fill or state) alongside Lucide Icons for structural UI shells in the PWA. On iOS, native SwiftUI renders corresponding system-configured SF Symbols to preserve platform-specific styling paradigms.

Platform Diversion: Icons

PWA / Web: Integrates Google Font-based Material Symbols Rounded for inline content variables and Lucide Icons rendered as local SVG vectors for clean interface grids.

iOS / Android: Discards web font files completely. Uses iOS-native system vector database SF Symbols (e.g. systemName: "mappin.circle") to match default Apple HIG specifications.

Grid Systems & The 26px Corner-Radius Specs

To deliver a premium native feel to the PWA web layer, we match Apple's iOS layout curvature spec. Grid items are arranged in fluid multi-column setups utilizing a signature, heavy 26px corner-radius (border-radius: 26px;) that groups content into solid spatial elements.

Trip Initialization

Captain specifies structural trip boundaries: When, Where, and Who. Sets coordinates of trip nodes.

Tactile Vibe Check

Tactile preferences completed silently before access is granted. Overlaps are graphed in area charts.

AI Decision Sweep

Google Gemini pulls relational datasets and processes itinerary routes to output high-resolution consensus.

Platform Diversion: Grid & Radius

PWA / Web: Content containers use standard CSS flexbox/grid layout and absolute border-radius declarations (e.g. border-radius: 26px;).

iOS / Android: iOS interfaces rely on SwiftUI dynamic stacks (VStack, LazyVGrid) and clip backgrounds using RoundedRectangle(cornerRadius: 26, style: .continuous) to get Apple's smooth squircle geometry.
