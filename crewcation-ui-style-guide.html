Introduction

Welcome to the interactive style guide for Crewcation. This guide documents the visual identity, tokens, and components that make up the Crewcation group travel framework.

Rescuing the Trip Captain

A dual-platform collaborative travel framework designed to align budgets, collect preferences, and execute flawless itineraries with your crew.

Purpose & Vision

Crewcation was designed to solve the structural chaos of planning group trips. By breaking down group action sequences into role-based workflows (Captain organizer vs Crew participant) and privately collecting travel styles (Vibe Check), Crewcation shifts the planning workspace from a cluttered WhatsApp thread into a structured collaborative workflow.

This Design System serves as the single source of truth for all designers and developers working on the ecosystem, maintaining consistent patterns across native iOS, responsive web PWAs, and native Android codebases.

UX Core Principles

  • Budgets are Private: Users specify budget brackets secretly. Crewcation's algorithm aggregates budget brackets and filters destinations automatically, avoiding social awkwardness.
  • Progressive Milestones: Trips move through structured phases: Create → Invite → Suggest → Vote → Execute. Features are locked or unlocked dynamically.
  • Platform-Specific Primitives: iOS uses Apple's native human interface guidelines (SF Symbols, default San Francisco typography, haptic feedbacks, squircle sheets). PWA uses fluid, lightweight web elements (Lucide icons, Inter fonts, warm forms) optimized for instant mobile web scanning.

Platform Adaptation Philosophy

Rather than enforcing rigid 1:1 pixel replication that breaks user expectations, Crewcation adapts design patterns to what users are familiar with. The iOS application feels fully integrated with iOS 18 because it leverages system-level components. The Next.js web application looks sleek and responsive, utilizing layout patterns designed for mobile browsers, ensuring that a user joining via web link gets a frictionless, high-quality onboarding experience.

Squircle & Inner Radii

iOS uses Apple's continuous corner curve — a superellipse that blends the straight wall into the corner arc much earlier than a standard circle. This makes corners feel smooth like a river stone, not geometric.

When nesting a rounded element inside another (e.g. a button inside a card), the inner corner radius must follow this rule:

rinner = router − p
where p = padding between parent edge and child element

Platform Icon Libraries

To maintain platform-appropriate ergonomics, the design system utilizes native icon frameworks instead of wrapping a single unified custom SVG sheet:

  • PWA / Web (Tailwind): Google Material Symbols Rounded (CDN stylesheet) and Font Awesome / Lucide icons (UI control elements).
  • iOS (SwiftUI): Native Apple SF Symbols database (integrated with SF font metrics).
  • Android (Jetpack Compose): Material Design 3 Rounded Icons pack (`androidx.compose.material.icons`).

Hybrid Implementation on Web

The Web PWA implements a hybrid system to optimize delivery. Layout controls (buttons, navigation headers, sidebar indicators) load structural vectors (Font Awesome / Lucide). Dynamic user data points (categories, trip tags, map markers) use Material Symbols Rounded to dynamically scale and support weight/fill attributes. For cross-platform parity, a lookup table maps web token values (e.g., location_on) to native identifiers (e.g., mappin.circle for iOS, Icons.Rounded.LocationOn for Android).

const SENTIMENT_STAGES = [
  { emoji: '😤', text: 'Hate It (1/10)' },
  { emoji: '😠', text: 'Dislike (2/10)' },

// Listening to user slider movement
const rangeInput = document.getElementById('demo-sentiment-input');
rangeInput.addEventListener('input', (e) => {
  const index = parseInt(e.target.value) - 1;
  const stage = SENTIMENT_STAGES[index];
  
  document.getElementById('demo-sentiment-emoji').innerText = stage.emoji;
  document.getElementById('demo-sentiment-label').innerText = stage.text;
});
// Gesture-based Custom Slider in SwiftUI
struct SentimentSlider: View {
    @Binding var rating: Int // 1 to 10
    
    var body: some View {
        GeometryReader { geometry in
            ZStack(alignment: .leading) {
                // Slider Track
                Capsule()
                    .fill(Color.gray.opacity(0.2))
                    .frame(height: 8)
                
                // Active Fill Track
                Capsule()
                    .fill(Color("ActionTeal"))
                    .frame(width: CGFloat(rating) / 10 * geometry.size.width, height: 8)
                
                // Draggable Thumb
                Circle()
                    .fill(Color.white)
                    .frame(width: 28, height: 28)
                    .shadow(radius: 4)
                    .offset(x: CGFloat(rating) / 10 * geometry.size.width - 14)
                    .gesture(
                        DragGesture()
                            .onChanged { value in
                                let percent = Double(value.location.x / geometry.size.width)
                                let newRating = Int((percent * 9.0).rounded()) + 1
                                let clamped = max(1, min(10, newRating))
                                if clamped != rating {
                                    rating = clamped
                                    triggerHapticFeedback()
                                }
                            }
                    )
            }
        }
        .frame(height: 30)
    }
    
    func triggerHapticFeedback() {
        let generator = UIImpactFeedbackGenerator(style: .light)
        generator.impactOccurred()
    }
}

Illustration Style Guide

All illustrations come from the Crewcation brand grain illustration library — a curated set of hand-crafted vector assets with a consistent grain texture treatment. The grain effect gives the illustrations warmth, depth, and a premium hand-crafted feel that differentiates Crewcation from generic flat-icon competitors.

Usage Rules

  • Background tinting: Always place illustrations on a soft, harmonious gradient background — never on plain white or dark surfaces. Use the brand palette tints (teal, coral, navy at 10–15% opacity).
  • Drop shadows: Apply a subtle drop-shadow(0 8px 16px rgba(38,40,71,0.12)) CSS filter to give illustrations visual lift without harsh edges.
  • Max size: Illustrations should never exceed 50% of their container width at desktop scale, and should be responsive with max-height: 200px in compact contexts.
  • No stretching: Always use object-fit: contain to preserve aspect ratios.
  • Alt text: All illustrations must have descriptive alt text for accessibility compliance.
